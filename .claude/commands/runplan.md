## 実装

1. `docs/tasks/` 配下の `04_implementation_plan.md` を読み、どのステップまで完了しているか確認する。
   凡例：`[ ]` 未着手、`[x]` 完了、`[-]` スキップ
   次の未着手フェーズを作業する。サブセクション（例: 2.1, 2.2）がある場合は、
   そのフェーズ全体を完了させてから次フェーズに進む。
2. 実装は `02_architecture.md` の設計に従う。
3. コード変更後は `make lint` と `make test` を実行し、エラーを修正する。
4. フェーズ完了ごとに計画書のチェックボックスを更新（完了 → `[x]`、スキップ → `[-]`）してから commit する。
5. 全フェーズ完了まで 1〜4 を繰り返す。完了したらレビューへ進む。

## レビュー

1. `make deadcode` を実行し、今回の変更で不要になった dead code がないか確認する。あれば削除して commit する。
2. 現在ブランチと派生元ブランチの差分に対してコードレビューを行う。確認観点：
   - `01_requirements.md` の受け入れ条件を全て満たしているか
   - `02_architecture.md` の設計と整合しているか
   - テストカバレッジは十分か（非自明なロジック・エラーパス・境界値がカバーされているか）
   - 既存テストとの重複がないか、自明すぎて価値のないテストがないか
   - 既存の関数・実装を使わず同等処理を自前実装していないか
   - コメントが全て英語か
   - コメントに AC-1 などの開発ドキュメント参照が残っていないか
3. 問題があれば修正して commit し、2 に戻る。問題がなければ PR 作成へ進む。

## PR 作成

1. 本ブランチの PR が GitHub 上にあれば `git push` して更新する。
2. PR がなければ `git push` して draft PR を作成する。
