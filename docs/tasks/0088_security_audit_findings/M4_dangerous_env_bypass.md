# M4: dangerous env value パターンがバイパス可能

- **重大度**: 🟡 Medium
- **領域**: 環境変数バリデーション (`internal/runner/security`)
- **影響コマンド**: `runner`

## 問題

[internal/runner/security/validator.go:152-174](../../../internal/runner/security/validator.go#L152-L174) 付近で、環境変数の **値** に対して「危険なパターン」を含まないかチェックしている。

```go
var dangerousValuePatterns = []string{
    ";", "|", "&", "$(", "`", ">", "<",
    "rm ", "sudo ", "chmod ", ...
}
```

### 検知対象の限界

1. **シェル非経由の exec**: 本プロジェクトは [executor.go:196](../../../internal/runner/executor/executor.go#L196) で `exec.CommandContext` を直接呼ぶため、環境変数の値がシェルで再解釈されることは基本的にない。したがって `;`, `|`, `$(` 等のシェルメタ文字を環境変数 **値** の側で検出しても、攻撃経路としては実際には閉じている (多層防御の一層にすぎない)。
2. **false negative (バイパス)**:
   - `rm ` (末尾スペース) は `rm\t`, `rm$IFS`, `/bin/rm`, `\rm` で回避可能。
   - `sudo ` 同様。
   - `$(` は `$ (` (スペース挿入) や bash の `${ ...;}` 形式で回避可能 (ただしシェル経由時のみ意味を持つ)。
3. **false positive**: `PATH=/usr/bin:/bin` のような正当値にも `:` は無いが、ログメッセージ等で `;`, `|` を含む値は実運用で普通に発生する (例: `USER_AGENT`, `MESSAGE`, JSON を環境変数で渡すケース)。

### 検知の意義

現状のパターンマッチは:

- **シェル経由で実行される別プロセス** (子コマンドが内部で system(3) を呼ぶなど) への対策としては意味がある。
- しかし本プロジェクト自身のコード経路で環境変数値がシェル解釈されることはない。
- 実質的な意義は「allowlist に通った値に明らかに怪しいペイロードが含まれていないか」という運用上のサニティチェック。

## 影響

- **直接的な影響なし**: シェル経由の再解釈が無い限り、これらのパターンが含まれていても即座に攻撃にはならない。
- **誤った安心感**: 「dangerous pattern をチェックしている」というアピールは、**実際にはシェル解釈される場面** (例: 子コマンドが bash スクリプトで、スクリプト内で `"$ENV_VAR"` を展開する) において不完全で、ユーザが油断する可能性がある。
- **正当値のブロック**: false positive により運用ノイズが発生する。

## 修正方針

以下の 3 案を比較検討すべき:

### 案 A: 削除 (検知なし)

- シェル非経由 exec なので値検知は本質的に不要。
- YAGNI 原則に則り、効果の薄いチェックを削除。allowlist による変数名制御に注力。
- **欠点**: 子プロセスが内部でシェルを呼ぶケース (multi-tier) でのガードが消える。

### 案 B: 現状維持 + ドキュメント

- 多層防御の 1 層として残す。
- ドキュメントで「これは万能ではなく、シェル経由実行を前提とする子プロセスへの追加保護」と明記。
- false positive を減らすためパターンを絞る。

### 案 C: 再設計 (厳格)

- 「制御文字 (`\n`, `\r`, `\0`) を含まない」等、**構文的** に検知可能な異常のみに絞る。
- シェルメタ文字の検知は allowlist 側で value パターン (regexp) を指定可能にすることで対応。

### 推奨

**案 C**。`\0` を含む値は execve に渡せないため即拒否は妥当。`\n` は環境変数値として glibc は受け付けるが、多くの子プロセスのパーサを混乱させるため拒否する合理性がある。現状の「シェル文字列マッチ」は本プロジェクトの実行モデルと噛み合っていない。
