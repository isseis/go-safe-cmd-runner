# 0091: Slack webhook URL のホスト allowlist

## 背景

[0089_security_audit_fixes](../0089_security_audit_fixes/01_requirements.md) のセキュリティ検査所見 L2 を独立タスクとして切り出したもの。

[internal/logging/slack_handler.go:132-152](../../../internal/logging/slack_handler.go#L132-L152) の `validateWebhookURL` は HTTPS スキームのみを検査し、ホスト名を制限しない。環境変数 (`GSCR_SLACK_WEBHOOK_URL_*`) が改ざんされた場合に、任意ホストへのログ送信 (情報漏洩・SSRF) が成立しうる。

---

## 設計方針

webhook URL 自体はポスト権限を持つ機密情報であるため、TOML 設定ファイルには書けない。一方でオンプレミスの Slack 互換サービスに対応するため、許可ホスト (ポリシー情報) のみを TOML に記載し、URL 本体は引き続き環境変数で管理する。

- **ポリシー (TOML)**: `global.slack_allowed_host` に許可する単一のホストを指定する。TOML はハッシュ検証対象であるため、改ざんを検出可能。
- **秘密情報 (環境変数)**: `GSCR_SLACK_WEBHOOK_URL_SUCCESS` / `GSCR_SLACK_WEBHOOK_URL_ERROR` は従来どおり環境変数で管理する。
- **検証フロー**: TOML から読んだ許可ホストと、環境変数の URL ホストが一致するかを検証する。`slack_allowed_host` が未設定の場合は検証エラーとなる。
- **ホスト名比較**: ポート番号を除いたホスト名で、大文字/小文字を区別せず完全一致で照合する。
- **起動順序**: 現状の起動フローでは Slack ハンドラ初期化が TOML 読み込みより先に行われるため、TOML 由来の許可ホストを使った検証が不可能である。bootstrap の起動フローを二段階に分割し、TOML 読み込み前はコンソール・ファイルハンドラのみを初期化し、TOML 読み込み後に許可ホストを用いた Slack ハンドラを追加する。
- **Slack 無効化の区別**:
  - 環境変数 (`GSCR_SLACK_WEBHOOK_URL_*`) が未設定 → Slack 通知をサイレントに無効化 (既存動作)
  - 環境変数が設定済み かつ `slack_allowed_host` が未設定 → ホスト検証エラーで起動失敗

---

## 受け入れ条件

**設定スキーマ:**

- AC-L2-1: TOML 設定ファイルの `[global]` セクションに Slack webhook URL の許可ホストを設定する項目 (`slack_allowed_host`) を追加すること
- AC-L2-2: Slack ハンドラ生成オプションに許可ホストを受け渡す設定を追加すること
- AC-L2-3: ロギング初期化オプションに許可ホストを受け渡す設定を追加すること
- AC-L2-4: ロガー設定に許可ホストを受け渡す設定を追加すること

**検証ロジック:**

- AC-L2-5: Slack ハンドラ生成時の webhook URL 検証に許可ホストによるホスト名検査を追加すること
- AC-L2-6: ホスト名の照合はポート番号を除いた形で行い、大文字/小文字を区別しない完全一致とすること
- AC-L2-7: 許可ホストが未設定の場合、すべての URL はホスト検証エラーを返すこと
- AC-L2-8: 許可ホストと一致しないホストの URL は検証エラーを返すこと
- AC-L2-9: 既存の HTTPS スキーム必須・ホスト名存在チェックを維持すること
- AC-L2-10: Slack ホスト検証エラーはログファイルエラーではなく設定エラーとして分類されること

**起動パイプライン:**

- AC-L2-11: Slack ハンドラの初期化は TOML 設定ファイルの読み込み完了後に行うこと
  - TOML 読み込み前: コンソールおよびファイルハンドラのみ初期化する
  - TOML 読み込み後: 許可ホストによる検証を経て Slack ハンドラを追加する
- AC-L2-12: TOML 設定から読み込んだ許可ホストが、ロギング初期化の各層を経由して Slack ハンドラ生成まで漏れなく伝播されること

**テスト:**

- AC-L2-13: 許可ホストが未設定の場合に URL 検証がエラーを返すことを確認するユニットテストを追加すること
- AC-L2-14: 許可ホストと一致しないホスト (例: `evil.example.com`) がエラーになることを確認するユニットテストを追加すること
- AC-L2-15: 許可ホストと一致するホスト (例: `hooks.slack.com`) の URL が検証を通過することを確認するユニットテストを含めること
- AC-L2-16: ホスト名の照合が大文字/小文字を区別しないことを確認するユニットテストを追加すること (例: `HOOKS.SLACK.COM` が `hooks.slack.com` の許可ホストで通過する)
- AC-L2-17: ポート番号付き URL (例: `https://hooks.slack.com:443/...`) が許可ホスト検証で正しく処理されることを確認するユニットテストを追加すること
- AC-L2-18: 既存の HTTPS スキーム検証・ホスト名存在チェックが引き続き機能することを確認するユニットテストを含めること
- AC-L2-19: TOML 設定の許可ホストが Slack ハンドラ生成まで正しく伝播されることを確認するユニットテストを追加すること
- AC-L2-20: 環境変数が設定済みかつ許可ホストが未設定の場合に、起動が設定エラーで失敗することを確認するテストを追加すること
